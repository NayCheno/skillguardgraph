# SkillGuardGraph Benchmark v0 — Data Card

## Dataset Name

**SkillGuardGraph Compositional Attack Benchmark v0**

## Version

v0.1 — Synthetic benchmark for evaluating cross-layer evidence fusion against compositional skill attacks, with realistic noise patterns.

## Description

A synthetic benchmark dataset for evaluating the detection of compositional attacks in AI agent skill ecosystems (e.g., MCP tool servers). Each sample is a JSON object representing a skill with its manifest, optional runtime trace, and labeled attack class. The benchmark targets seven compositional attack categories that exploit gaps across metadata, implementation, runtime, and governance layers — attacks that no single-layer defense reliably detects.

## Sources

All data is synthetically generated by template-based randomization (`experiments/scripts/build_benchmark.py`). No real-world tool manifests, credentials, or production data are included. Skill names, descriptions, publishers, and traces are procedurally constructed from curated vocabulary pools.

## Size and Composition

| Class | Count | Description |
|---|---|---|
| benign | 1,000 | Benign skills: 700 clean (read-only, trusted) + 300 noisy (write scopes, untrusted publishers, file ops — legitimately justified) |
| capability_laundering | 430 | T1: Declares read-only but requests write/export/admin scope (258 standard + 172 subtle) |
| cross_skill_confused_deputy | 430 | T2: Untrusted source flows to high-privilege tool (258 standard + 172 subtle) |
| delayed_rug_pull | 430 | T3: Approved version drifts to high-risk behavior (258 standard + 172 subtle) |
| consent_laundering | 430 | T4: Approval text derived from untrusted context (258 standard + 172 subtle) |
| persistence_pivot | 430 | T5: Untrusted source writes to persistent store (258 standard + 172 subtle) |
| split_exfiltration | 430 | T6: Sensitive read → transform → external write chain (258 standard + 172 subtle) |
| scope_inflation | 430 | T7: Task needs read but granted write/delete/export (258 standard + 172 subtle) |
| **Total** | **4,010** | |

Each malicious sample includes a `benign_pair` field referencing a benign counterpart for paired evaluation (FPR, task success impact).

## Label Schema

```json
{
  "label": "benign | capability_laundering | cross_skill_confused_deputy | delayed_rug_pull | consent_laundering | persistence_pivot | split_exfiltration | scope_inflation"
}
```

Labels are mutually exclusive. A sample's `label` always matches its `attack_class` field (or `"benign"` with no `attack_class`).
## Safety Controls

| Control | Implementation |
|---|---|
| No real credentials | All signatures are synthetic (`sig-{name}-{N}`). No API keys, tokens, or passwords. Validated by automated regex scan. |
| Sinkhole domains only | All external endpoints use `*.sinkhole.test` or `*.example.invalid` TLDs. No real URLs in traces or source code. |
| Synthetic data only | All skill names, descriptions, publishers, and trace data are procedurally generated. No real-world tool manifests. |
| Reproducible | Generation uses a fixed random seed (42). Re-running the build script produces identical output. |
| No executable payloads | Source code stubs are pseudocode comments, not runnable code. Runtime traces are JSON-structured event logs. |

## Schema

Formal JSON Schema: `experiments/data/schema/skill_schema.json` (draft-07).

## File Format

- `samples.jsonl`: One JSON object per line. Each line is a complete, self-contained sample.
- `stats.json`: Aggregate counts per class, generation parameters.

## How to Use

```bash
# Generate the dataset
cd experiments
PYTHONPATH=src python scripts/build_benchmark.py

# Validate structural invariants
PYTHONPATH=src python scripts/validate_labels.py
```

Load in Python:

```python
import json

samples = []
with open("experiments/data/benchmark_v0/samples.jsonl") as f:
    for line in f:
        samples.append(json.loads(line))
```

## Known Limitations

1. **Synthetic only.** Samples are template-generated, not sourced from real-world tool ecosystems. Attack patterns may lack the diversity and subtlety of genuine adversarial behavior.
2. **Template uniformity.** Vocabulary pools are finite (~50 skill names, ~10 description templates). Repeated patterns may leak through to models that memorize surface features.
3. **No multi-turn traces.** Traces are single-session event sequences. Cross-session persistence attacks (T5) are represented by static markers, not simulated multi-session execution.
4. **Benign-pair coupling.** Benign pairs are randomly assigned, not crafted to be maximally similar to their malicious counterpart. A production benchmark would pair by function (e.g., benign PDF summarizer ↔ malicious PDF summarizer).
5. **No code analysis ground truth.** Source code stubs are present on ~50% of capability_laundering and delayed_rug_pull samples. Static analysis baselines would benefit from fuller coverage.
6. **Seven classes only.** The taxonomy covers compositional attacks. Single-layer attacks (prompt injection without composition, naive over-permissioning) are out of scope.

## Intended Use

- **Primary:** Evaluating the SkillGuardGraph cross-layer evidence fusion system against compositional attack detection.
- **Secondary:** Benchmarking single-layer baselines (metadata scanner, LLM judge, static analysis, sandbox prober, HITL-only policy) against the same taxonomy.
- **Not intended for:** Training ML models (the dataset is too small and synthetic), real-world threat intelligence, or as a substitute for production security tooling.

## Ethics Statement

This dataset is designed for defensive security research. All attack patterns are synthetic and serve to evaluate detection capabilities. The benchmark:

- Contains no real malware, exploits, or attack tools.
- Uses no real credentials, personal data, or production system references.
- Employs sinkhole domains to prevent any accidental interaction with real services.
- Is released to improve the security of AI agent ecosystems, not to enable attacks.

Researchers using this benchmark should ensure their evaluation environments are appropriately isolated and that any models trained or evaluated on this data are used solely for defensive purposes.
